# 拯救微机原理

# 1 . 7种寻址方式中操作数的位置

  • 立即寻址：操作数作为指令的一部分而直接写在指令中，这种操作数称为立即数，立即数（操作数）紧跟在操作码之后
  • 寄存器寻址：操作数存储在某寄存器中，或把目标操作数存入寄存器
  • 直接寻址方式：指令所要的操作数存放在内存中，在指令中直接给出该操作数的有效位置
  • 寄存器间接寻址方式：操作数在寄存器中，操作数的有效地址用SI、DI、BX和BP等四个寄存器之一来指定
  • 寄存器相对寻址：操作数在存储器中，其有效地址是一个基址寄存器(BX、BP)或变址寄存器(SI、DI)的内容和指令中的8bit、16bit偏移量之和
  • 基址加变址寻址方式：操作数在存储器中，其有效地址是一个基址寄存器(BX、BP)和一个变址寄存器（SI、DI）的内容之和
  • 相对基址加变址寻址方式：操作数在存储器中，其有效地址是一个基址寄存器(BX、BP)的值、一个变址寄存器（SI、DI）的值和指令中的8bit、16bit偏移量之和

  • 硬件系统：运算器、存储器、输入与输出设备和控制器（四大部分）

# 3. 8086的BIU与EU的组成与作用是什么？

  • BIU(Bus Interface Unit)：总线接口单元，由段寄存器、指令指针寄存器IP、指令队列、地址加法器以及总线控制逻辑组成；负责CPU与存储器、I/O接口之间的信息 （包括数据、地址、控制命令等）传送， 包括生成访问存储器所需的20位的物理地址、不断从内存中取指令并送到指令队列、以及配合EU对指定的内存单元或者外设端口进行数据存取操作

  • EU(Execution Unit)：执行单元，由算术逻辑运算单元ALU、暂存器、标志寄存器（即PSW寄存器）、通用寄存器组和EU控制器构成；其作用为执行指令，进行全部算术逻辑运算、完成偏移地址的计算，向BIU提供指令执行结果的数据和访问存储器需要的偏移地址，并对通用寄存器和标志寄存器进行管理

# 6. 指令操作数的寻址方式的判断

参考第一题

  • 立即寻址：操作数直接放在指令里，如 MOV AX, 3

  • 寄存器寻址：操作数放在某个寄存器中，如 MOV AX,BX

  • 直接寻址：操作数的地址直接放在指令里，如 MOV AX, [0010H]

  • 寄存器间接寻址：和寄存器寻址的区别是，BX里存的不是操作数，而是操作数的地址。如 MOV AX,[BX]

  • 寄存器相对寻址：其有效地址是一个基址寄存器(BX、BP)或变址寄存器(SI、DI)的内容和指令中的8bit、16bit偏移量之和，如：MOV AX [BX+100H]

  • 基址加变址寻址方式：其有效地址是一个基址寄存器(BX、BP)和一个变址寄存器（SI、DI）的内容之和，如：MOV BX,[BX+SI]

  • 相对基址加变址寻址方式:其有效地址是一个基址寄存器(BX、BP)的值、一个变址寄存器（SI、DI）的值和指令中的8bit、16bit偏移量之和,如MOV AX, [BX+SI+200H]

# 7. 堆栈的概念、push、pop指令的含义、sp的含义

堆栈：以**后进先出(Last In First Out)**的规则存取信息的一种存储机构

push指令：进栈指令，将信息送入堆栈；书上原句为：将SRC或标志寄存器FLAG的内容送入堆栈，并修正堆栈指针的内容。

pop指令：出栈指令，从堆栈中取出信息；书上原句为：将SP所指出的栈顶内容取出，并送入DST所指寄存器、内存某单元或标志寄存器FLAG，并修正SP内容。

SP：SP为堆栈指针寄存器，用来存放堆栈区的偏移地址。时刻指向栈顶。

# 11.偏移地址、逻辑地址、物理地址的概念及计算

# 17. 8086一个总线周期的构成，至少有多少个时钟周期，CPU通过检测什么引脚的信号来决定是否增加TW等待周期。TW等待周期插在哪里？

一个总线周期至少有4个时针周期；CPU通过检测READY引脚信号决定是否增加TW等待周期，若READY为低电平，则表示系统中挂接的存储器或外设不能如期送出数据，要求CPU在T3周期和T4周期之间插入一个或者几个等待周期Tw，如果READY为高电平，则进入T4周期。

# 18.SRAM、DRAM的区别是什么？

DRAM:动态随机存取存储器，需要不断的刷新，才能保存数据.而且是行列地址复用的，许多都有页模式

SPAM: 静态的随机存取存储器，加电情况下，不需要刷新，数据不会丢失，而且，一般不是行列地址复用的

# 22. 微型计算机数据传递的控制方式主要有哪几种？各有什么特点？

主要有四种：

  • 程序控制方式：CPU与外设间数据传送是在程序的控制下完成的一种数据传送方式；此方式程序设计简单，硬件软件较省，但费时，CPU效率低，实时性差。主要用于中低速外设和实时性要求不高的场合

  • 中断控制方式：实时性好，不需要反复查询等待，减少cpu等待时间，cpu与外设可并行工作

  • DMA方式：不需cpu介入，减少了cpu的开销，能实现高速数据块传送，提高了效率，但增加了硬件开销，提高了系统的成本

  • IOP方式：传送速度快且不需cpu介入，可与cpu并行工作，效率高

# 35. RAM与ROM的主要区别有哪些？

# 43.指令 MOV AX, DX 和 MOV AX, [SI]中，执行速度较快的是?

MOV AX, DX是寄存器寻址，而MOV AX, [SI]是寄存器间接寻址，而寄存器寻址是直接将操作数放在寄存器中，寄存器间接寻址则是将操作数地址放在寄存器中，操作数保存在另一个寄存器中，故显然寄存器寻址要更快一些。

# 44. 调用子程序指令及转移指令的执行过程主要有什么区别？

# 45. CMOS RAM和BIOS ROM各是什么含义？二者之间是什么关系？

CMOS RAM全称为Real Time Clock and CMOS RAM，它是主机板上完成特殊功能的RAM存储器（它实际由两部分电路组成：实时时钟电路和256字节的RAM，用于存储实时时间信息和整机系统的配置信息）

BIOS ROM：主机板上完成特殊功能的ROM存储器（包含两大程序模块：开机加电后直接运行的部分，用户可以调用的部分）

BIOS中的系统设置程序是完成CMOS参数设置的手段；CMOS RAM既是BIOS设定系统参数的存放场所，又是BIOS设定系统参数的结果。总结：通过BIOS设置程序对CMOS参数进行设置

# 49. 外部硬中断分哪几种类型？它们的区别是什么？外部硬中断和子程序调用有什么区别？

可分为：

  • 可屏蔽中断：受IF位控制的中断，如果IF=1，微处理器可以屏蔽中断请求，称为开中断状态，如果IF位为0，就关闭了微处理器响应可屏蔽中断的功能，称为关中断状态；
  • 不可屏蔽中断：不能被CPU用指令CLI（IF位置0）来禁止，一旦出现请求，CPU不可屏蔽；

外部硬中断和子程序调用有什么区别：

  • 子程序调用中，什么时候执行子程序都是由程序员自己写（可控）的，而外部硬中断的发生时间和位置我们是不知道的，如果说调用子程序是由主程序主动发起的，那么，进入中断服务程序必须由中断源主动申请，即由中断服务程序主动发起
  • 子程序调用每次只能调用唯一一个子程序，但是在中断系统中往往有多个中断源同时申请中断服务，即有多个中断服务程序同时要求执行的情况发生
  • 子程序通常与调用它的主程序或上一层子程序之间有非常紧密的联系，而中断服务程序一般与被中断的现行程序之间没有关系

# 50. 对于8086，若当(CS)=2020H时，物理地址为24200H，则当(CS)=6520H时，物理地址应移动到什么地方？此时(IP)=？

故可知偏移地址=物理地址-段地址*10H = 24200H-20200H = 4000H

当CS = 6520H时，物理地址 = 65200H+4000 = 69200H

而IP是指令指针寄存器，是一个16位寄存器，用来存放代码段中的偏移地址，故（IP）=偏移地址 = 4000H

# 54. 8086微处理器是如何实现地址总线和数据总线分时复用的？

首先了解一下分时复用：就是一段时间用做地址总线，一段时间用做数据总线；

如何实现：每次执行存储器读写或对I/O端口进行输入输出操作时，都要用一个总线周期时间，每个总线周期由几个时钟周期T组成，在总线周期的T1期间，总线的低16位作为地址总线，输出地址码A15～A0；而在其他时钟周期T期间，作为双向数据线D15～D0，输入或输出16位数据；

# 55. CPU与外围设备之间的数据传输要使用接口电路的主要原因是什么？

因为CPU每次只能访问一个外设，因此并行信息的外设需增加选通功能，才能满足CPU的访问要求，必须用接口电路；
